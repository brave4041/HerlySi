01 MIXINS & MULTIPLE INHERITANCE
DARTPAD

Dart tidak mendukung multiple inheritance tradisional (satu anak banyak induk). Sebagai solusinya, Dart menggunakan Mixins untuk berbagi perilaku antar class tanpa hierarki yang rumit.

  • Gunakan keyword mixin untuk mendefinisikan perilaku.
  • Gunakan keyword with untuk menerapkan mixin ke class.
  • Class dapat menggunakan banyak mixin sekaligus.
DART // MIXIN DASAR 
mixin CanFly {
  void fly() {
    print('Can fly!');
  }
}

mixin CanSwim {
  void swim() {
    print('Can swim!');
  }
}

class Bird with CanFly {}
class Duck extends Bird with CanSwim {}

void main() {
  print("--- TEST BASIC MIXIN ---");
  var b = Bird(); b.fly();
  var d = Duck(); d.fly(); d.swim();
}

02 MIXIN DENGAN STATE
DARTPAD

Mixin juga bisa memiliki properti (state) dan getter/setter. Ini sangat berguna untuk melacak status perilaku objek secara modular.

DART // STATEFUL MIXIN 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

mixin CanFlyState {
  bool _isFlying = false;
  bool get isFlying => _isFlying;
  
  void fly() {
    if (!_isFlying) { 
      _isFlying = true; 
      print('Started flying!'); 
    } else { 
      print('Already flying!'); 
    }
  }
}

class BirdWithState with CanFlyState {}

void main() {
  print("--- TEST MIXIN WITH STATE ---");
  var bird = BirdWithState();
  bird.fly(); 
  bird.fly();
  print('Is Bird flying? : ${bird.isFlying}');
}

03 SEALED CLASS
VS CODE

Keyword sealed digunakan untuk membuat hierarki class yang tertutup. Subclass hanya boleh ada di file yang sama (library yang sama). Sering disebut sebagai Algebraic Data Types.

  • Definisi Utama: Menjamin bahwa tidak ada pihak luar yang bisa menambah subclass secara ilegal. Semua variasi class harus didefinisikan di tempat yang sama (Library-Locked).
  • Keunggulan: Fitur Exhaustive Checking, di mana compiler akan memberikan Error jika ada tipe subclass yang belum ditangani di switch. Ini sangat berguna untuk manajemen state (Loading, Success, Error).
    • KESIMPULAN (SESUAI SLIDE):
    • Exhaustive switch: Compiler memastikan semua status (Success/Error) tertangani.
    • Type-safe: Tidak akan ada runtime error karena case yang terlupa.
    • Pattern matching: Kita bisa langsung ambil field data di dalam switch.
    • Clear hierarchy: Subclass terkunci hanya di dalam library ini saja.

import 'libsealed.dart';

void handleResponse(ApiResponse response) {
  switch (response) {
    case Success(:final message): print("Sukses: $message");
    case Error(:final errorMessage): print("Error: $errorMessage");
  }
}

void handleStatus(AppStatus status) {
  switch (status) {
    case Loading(): print('⏳ Sedang memuat...');
    case Loaded(:final data): print('✅ Data Berhasil: $data');
    case Failed(:final error): print('❌ Gagal: $error');
  }
}

void main() {
  print("--- TEST SEALED CLASS & SIMULASI STATUS ---");
  
  handleResponse(Success("Koneksi berhasil!"));
  handleResponse(Error("Error 404: File tidak ditemukan"));

  print("\n[SIMULASI DAFTAR STATUS]");
  final states = [
    Loading(),
    Loaded('Katalog Produk'),
    Failed('Server sibuk (500)'),
  ];

  for (var i = 0; i < states.length; i++) {
    print("State ke-${i + 1}:");
    handleStatus(states[i]);
  }
}

04 BASE CLASS
VS CODE

Keyword base digunakan untuk menegakkan aturan inheritance yang ketat. Modifier ini memastikan bahwa kode lu tetap memiliki kontrak yang jelas melalui pewarisan.

  • Definisi Utama: Class dapat di-Extend (pewarisan), namun dilarang keras di-Implement sebagai Interface di luar library. Ini menjamin bahwa setiap subclass selalu memiliki implementasi dasar yang sama.
  • Kapan Menggunakan: Digunakan saat lu ingin mengontrol bagaimana class diwariskan untuk menjaga API Stability. Menjamin bahwa struktur internal class tidak dirombak sembarangan oleh user lain.
  • Subclass Requirement: Setiap class yang mewarisi base class wajib ditandai kembali menggunakan base, final, atau sealed.
ANALISIS KONSEP SEPARASI
  • Lib Side (libbase.dart): Menetapkan "Pondasi Dasar" yang mewajibkan pewarisan (Extend).
  • Esensi: Modifer base menjaga kestabilan API agar struktur internal class tetap utuh.
  • Error Scenario: Jika kamu mencoba class X implements Animal dari luar library, akan muncul error: "The class 'Animal' can't be implemented outside of its library because it's a base class."

05 FINAL CLASS
VS CODE

Keyword final adalah modifier yang memberikan keamanan total (*Security by Closure*). Lu menutup rapat akses modifikasi dari luar.

  • Definisi Utama: Di luar library asal, class ini dilarang di-extend maupun di-implement. Class ini benar-benar berdiri sendiri dan tertutup (Closed for modification).
  • Encapsulation: Menjamin bahwa logika di dalam class tersebut tidak akan pernah berubah atau "disimpangkan" oleh subclass lain yang mungkin merusak sistem.
  • Subclass Requirement: Jika lu melakukan inheritance di dalam file yang sama, subclass-nya pun harus ditandai base, final, atau sealed.
ANALISIS KONSEP SEPARASI
  • Lib Side (libfinal.dart): Mengunci class sepenuhnya dari modifikasi eksternal.
  • Esensi: Memberikan keamanan level tinggi di mana logika bisnis asli tidak bisa di-override sama sekali.
  • Error Scenario: Mencoba extends Settings akan memicu error: "The class 'Settings' can't be extended, implemented, or mixed in outside of its library because it's a final class."

final class Settings {
  final String theme;
  final int fontSize;
  Settings({required this.theme, required this.fontSize}) {
    if (fontSize <= 0) {
      throw ArgumentError("Font size harus lebih besar dari 0!");
    }
  }
  void showSettings() { print("Theme: $theme, Font: $fontSize"); }
}

ESENSI CLASS MODIFIER

Kenapa kita harus menggunakan Class Modifier?

Esensi dari modifier seperti base, final, dan sealed adalah keamanan dan kontrol library. Aturan pembatasan (seperti dilarang meng-implement) TIDAK AKAN berlaku jika semua code digabung dalam satu file.

  • Library Boundary: Dart hanya menegakkan aturan modifier saat diakses dari luar library (file yang berbeda).
  • Predictability: Developer bisa menjamin bahwa class miliknya tidak akan disalahgunakan oleh orang lain yang mengimport library-nya (API Stability).
  • Encapsulation: Menavoid "leak" implementasi yang tidak diinginkan dari pihak ketiga.

06 INTERFACE CLASS
VS CODE

Keyword interface digunakan untuk mendefinisikan kontrak yang bisa di-Implement oleh class lain, namun dilarang keras untuk di-Extend (wariskan) di luar library.

  • Total Implementation: Pihak luar wajib mengimplementasikan ulang semua metode jika ingin menggunakan class ini sebagai interface.
  • Instantiable: Berbeda dengan abstract, interface class tetap bisa dibuatkan objeknya secara langsung.
  • API Security: Menjamin bahwa struktur class lu tidak akan di-override atau dirombak logikanya melalui pewarisan.
ANALISIS KONSEP SEPARASI
  • Esensi: Mencegah pewarisan paksa (inheritance) yang bisa merusak enkapsulasi internal library.
  • Validasi Error: Mengimplementasikan pengecekan fuel sebagai syarat validitas fungsional (Skenario Error jika habis).
  • Error Scenario: Mencoba extends Vehicle akan memicu error: "The class 'Vehicle' can't be extended outside of its library because it's an interface class."

08 SEALED CLASS + PATTERN MATCHING
DARTPAD

Kombinasi terbaik di Dart Modern (3.0+). Compiler akan Error jika ada case yang terlewat (Exhaustive Switch).

DART // STATE MANAGEMENT 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

sealed class AppState {}
class Loading extends AppState {}
class Success extends AppState { final List<String> items; Success(this.items); }
class Failed extends AppState { final String msg; Failed(this.msg); }

void updateUI(AppState state) {
  switch (state) {
    case Loading(): print('⏳ Loading...');
    case Success(:final items): print('✅ Data: $items');
    case Failed(:final msg): print('❌ Error: $msg');
  }
}

void main() {
  updateUI(Success(['Dart', 'Flutter', 'Modern']));
  updateUI(Failed('Connection Lost'));
}
