01 MIXINS & MULTIPLE INHERITANCE
DARTPAD

Dart tidak mendukung multiple inheritance tradisional (satu anak banyak induk). Sebagai solusinya, Dart menggunakan Mixins untuk berbagi perilaku antar class tanpa hierarki yang rumit.

  • Gunakan keyword mixin untuk mendefinisikan perilaku.
  • Gunakan keyword with untuk menerapkan mixin ke class.
  • Class dapat menggunakan banyak mixin sekaligus.
DART // BASIC MIXIN 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

mixin CanFly {
  void fly() {
    print('Can fly!');
  }
}

mixin CanSwim {
  void swim() {
    print('Can swim!');
  }
}

class Bird with CanFly {}
class Duck extends Bird with CanSwim {}

void main() {
  var b = Bird(); b.fly();
  var d = Duck(); d.fly(); d.swim();
}

02 MIXIN DENGAN STATE
DARTPAD

Mixin juga bisa memiliki properti (state) dan getter/setter. Ini sangat berguna untuk melacak status perilaku objek secara modular.

DART // STATEFUL MIXIN 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

mixin CanFly {
  bool _isFlying = false;
  bool get isFlying => _isFlying;
  void fly() {
    if (!_isFlying) { _isFlying = true; print('Started flying!'); }
    else { print('Already flying!'); }
  }
}

class Bird with CanFly {}

void main() {
  var bird = Bird();
  bird.fly(); 
  print('Is Bird flying? : ${bird.isFlying}');
}

03 SEALED CLASS
VS CODE

Keyword sealed digunakan untuk membuat hierarki class yang tertutup. Subclass hanya boleh ada di file yang sama (library yang sama).

  • Mencegah ekstensi dari luar library.
  • Memungkinkan compiler melakukan Exhaustive Checking pada switch/case.
  • Sangat cocok untuk pemodelan Result atau API Response.
DART // SEALED CLASS 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is Error) {
    print("Error: ${response.errorMessage}");
  }
}

void main() {
  handleResponse(Success("Data fetched!"));
}

04 BASE CLASS
VS CODE

Keyword base memastikan bahwa class hanya bisa di-extend, tidak boleh di-implement dari luar library. Ini untuk menjaga kontrak inheritance tetap solid.

DART // BASE CLASS 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

base class Animal {
  void speak() { print("Some sound..."); }
}

base class Dog extends Animal {
  @override
  void speak() { print("Woof!"); }
}

void main() {
  Animal dog = Dog();
  dog.speak();
}

05 FINAL CLASS
VS CODE

Keyword final menutup class sepenuhnya dari luar library. Class tidak bisa di-extend maupun di-implement di luar file penyusunnya.

DART // FINAL CLASS 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

final class Settings {
  final String theme;
  Settings(this.theme);
  void show() => print("Theme: $theme");
}

void main() {
  var s = Settings("Cyberpunk Dark");
  s.show();
}

07 SEALED CLASS + PATTERN MATCHING
DARTPAD

Kombinasi terbaik di Dart Modern (3.0+). Compiler akan Error jika ada case yang terlewat (Exhaustive Switch).

DART // STATE MANAGEMENT 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : Si2A

sealed class AppState {}
class Loading extends AppState {}
class Success extends AppState { final List<String> items; Success(this.items); }
class Failed extends AppState { final String msg; Failed(this.msg); }

void updateUI(AppState state) {
  switch (state) {
    case Loading(): print('⏳ Loading...');
    case Success(:final items): print('✅ Data: $items');
    case Failed(:final msg): print('❌ Error: $msg');
  }
}

void main() {
  updateUI(Success(['Dart', 'Flutter', 'Modern']));
  updateUI(Failed('Connection Lost'));
}
