01 PENGANTAR EXTENSION METHODS

Extension Methods (tersedia mulai Dart 2.7) adalah fitur canggih yang memungkinkan kita untuk menambahkan metode baru (fungsi, getter, setter, atau operator) ke tipe data yang sudah ada (baik tipe built-in maupun kelas buatan sendiri) tanpa memodifikasi kode asli dari tipe tersebut atau membuat kelas turunan baru.

METAFORA & PRINSIP UTAMA:
  • Analog: Memberikan kemampuan baru pada benda yang sudah ada tanpa harus memodifikasi struktur internal benda tersebut.
  • Tanpa Modifikasi: Solusi terbaik ketika kita ingin menambah utility pada SDK Dart atau package pihak ketiga yang kodenya tidak bisa kita ubah.
  • Type-Safe: Didukung penuh oleh static analysis compiler Dart untuk mendeteksi kesalahan tipe data saat compile-time.

02 MASALAH TANPA EXTENSION METHODS

Sebelum adanya Extension Methods, developer biasanya membuat Utility Class statik untuk memanipulasi tipe data. Cara ini menghasilkan penulisan kode yang kurang natural (tidak berorientasi objek).

class StringUtils {
  static String capitalize(String text) {
    if (text.isEmpty) return text;
    return text[0].toUpperCase() + text.substring(1);
  }

  static bool isEmail(String text) {
    return text.contains('@');
  }
}

  print('Original name: $name');
  print('Capitalized name: $capitalized');
  print('Is "test@mail.com" an email? $isEmail');
}

extension NumberExtensions on num {
  // Format ke Rupiah
  String toRupiah() {
    return 'Rp${toStringAsFixed(0).replaceAllMapped(
      RegExp(r'(\d{1,3})(?=(\d{3})+(?!\d))'),
      (match) => '${match[1]}.',
    )}';
  }

  // Check jika dalam range
  bool isBetween(num min, num max) {
    return this >= min && this <= max;
  }

  // Pangkat
  num power(int exponent) {
    var result = this;
    for (var i = 1; i < exponent; i++) {
      result *= this;
    }
    return result;
  }
}

extension ListExtensions<T> on List<T> {
  // Safe get dengan default null jika index di luar batas
  T? getOrNull(int index) {
    if (index < 0 || index >= length) return null;
    return this[index];
  }

  // Membagi list menjadi beberapa bagian (chunks)
  List<List<T>> chunk(int size) {
    var chunks = <List<T>>[];
    for (var i = 0; i < length; i += size) {
      chunks.add(sublist(i, i + size > length ? length : i + size));
    }
    return chunks;
  }

  // Mengambil elemen unik saja
  List<T> distinct() {
    var result = <T>[];
    for (var item in this) {
      if (!result.contains(item)) {
        result.add(item);
      }
    }
    return result;
  }
}

}

extension TextUtilsExtensions on String {
  String capitalize() => '${this[0].toUpperCase()}${substring(1).toLowerCase()}';
}

void main() {

extension DoubleRupiahFormatter on double {
  String toRupiah() {
    final intValue = toInt();
    String numberString = intValue.toString();
    String formattedString = '';
    int counter = 0;
    for (int i = numberString.length - 1; i >= 0; i--) {
      formattedString = numberString[i] + formattedString;
      counter++;
      if (counter % 3 == 0 && i != 0) {
        formattedString = '.' + formattedString;
      }
    }
    return 'Rp$formattedString';
  }
}

extension ProductExtensions on Product {
  Product withDiscount(double percentage) {
    var discountedPrice = price * (1 - percentage / 100);
    return Product(name, discountedPrice);
  }

  String get displayInfo => '$name - ${price.toRupiah()}';

  bool get isExpensive => price > 1000000;
}

extension StringValidators on String {
  bool get isEmail {
    return RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$').hasMatch(this);

  bool get isPhoneNumber {

        contains(RegExp(r'[a-z]')) &&

    } catch (_) {
      return false;
    }
  }
}

13 RINGKASAN & BEST PRACTICES

Berikut adalah pedoman penulisan Extension Methods yang baik agar kode tetap rapi dan terorganisir:

BEST PRACTICES:
  • Penamaan Jelas: Berikan nama deskriptif seperti StringExtensions atau DateTimeUtils agar mudah dibaca di IDE.
  • Organisasi File: Kelompokkan extension berdasarkan tipe targetnya, simpan satu tipe per file library.
  • Jangan Overuse: Hindari menggunakan extension method untuk setiap helper utility, gunakan hanya bila utility tersebut memperjelas interaksi data objek.
  • Null Safety: Dukung tipe data nullable (dengan tanda tanya ?) secara eksplisit jika method berpotensi dipanggil pada nilai null.

BONUS SOAL 1: KONFLIK NAMA ANTAR EXTENSION
SOAL BONUS 1

Kasus: Diberikan dua extension pada tipe data String yang mendefinisikan method toInt() dengan tipe kembalian berbeda.

FILE: test_bonus_1.dart 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : SI2A

extension StringNumber on String {
  int toInt() {
    return int.parse(this);
  }
}

extension StringDouble on String {
  double toInt() {  // ← perhatikan nama method sama, return type beda
    return double.parse(this);
  }
}

void main() {
  String angka = '123';
  
  // Pemanggilan langsung di bawah ini akan menyebabkan compile error
  // print(angka.toInt());
  
  // Solusi A: Pemanggilan Eksplisit
  int hasilInt = StringNumber(angka).toInt();
  double hasilDouble = StringDouble(angka).toInt();

  print('Pemanggilan Eksplisit StringNumber: $hasilInt (Tipe: ${hasilInt.runtimeType})');
  print('Pemanggilan Eksplisit StringDouble: $hasilDouble (Tipe: ${hasilDouble.runtimeType})');
}
ANALISIS JAWABAN SOAL 1:

1. Apa yang terjadi saat kode asli dijalankan?
Kode akan gagal dikompilasi (Compile-time Error). Pesan kesalahan akan menunjukkan adanya ambiguitas (ambiguous member lookup) karena compiler kebingungan menentukan method toInt() milik extension mana yang harus dieksekusi.

2. Mengapa hal itu terjadi?
Kedua extension (StringNumber dan StringDouble) memperluas tipe target yang sama (String) dan mendefinisikan method dengan nama yang persis sama (toInt()). Meskipun return type mereka berbeda (int vs double), Dart tidak mendukung overload method berdasarkan tipe kembalian maupun signature parameter dalam pemanggilan extension implisit.

3. Solusi:
• Pemanggilan Eksplisit: Bungkus variabel target dengan nama extension yang diinginkan secara eksplisit, misalnya: StringNumber(angka).toInt() dan StringDouble(angka).toInt().
• Menggunakan import prefix / hide: Jika didefinisikan di file terpisah, kita bisa menggunakan kata kunci hide saat import untuk menyembunyikan salah satu extension, atau memberinya alias prefix (as) dan memanggilnya secara spesifik.

BONUS SOAL 2: MEMBER PRIVATE & EXTENSION
SOAL BONUS 2

Kasus: Extension method mencoba mengakses variabel/member private (diawali underscore _) yang dideklarasikan di dalam kelas target.

FILE: test_bonus_2.dart 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : SI2A

// Solusi A: Menaruh class dan extension di library (file) yang sama
class Counter {
  int _value = 0;  // private pada tingkat library

  void increment() {
    _value++;
  }

  // Solusi B (Alternatif): Membuat getter public jika extension diletakkan di file terpisah
  int get value => _value;
}

extension CounterStats on Counter {
  int getValue() {
    // Karena extension ini berada di file yang sama dengan class Counter,
    // baris di bawah ini TIDAK BERMASALAH dan sukses dikompilasi.
    return _value; 
    
    // Jika di file terpisah, ganti dengan: return value;
  }
}

void main() {
  var c = Counter();
  c.increment();
  print('Hasil getValue(): ${c.getValue()}');
}
ANALISIS JAWABAN SOAL 2:

1. Mengapa kode asli gagal dikompilasi jika diletakkan di file terpisah?
Di Dart, akses modifier private (diawali dengan _) bersifat private pada level library (file), bukan level class. Apabila extension CounterStats dideklarasikan di file terpisah dari class Counter, extension tersebut dianggap berada di luar library class Counter sehingga tidak diizinkan mengakses variabel private _value.

2. Cara memperbaiki tanpa mengubah visibilitas _value menjadi public:
• Solusi A (Same File): Pindahkan deklarasi extension CounterStats ke dalam file yang sama dengan class Counter. Dengan begitu, compiler mengizinkan akses ke member private karena masih dalam library yang sama.
• Solusi B (Getter Public): Sediakan getter public di dalam class Counter (misalnya: int get value => _value;) untuk memaparkan nilainya secara read-only tanpa membiarkan pihak luar memodifikasi nilainya secara langsung. Kemudian, panggil getter tersebut dari extension (return value;).

BONUS SOAL 3: PRIORITAS METHOD ASLI VS EXTENSION
SOAL BONUS 3

Kasus: Kelas memiliki method asli (instance method) dengan nama yang sama dengan method yang didefinisikan dalam extension.

FILE: test_bonus_3.dart 
// Nama   : Herly Afrizal
// Nim    : 251410002
// Kelas  : SI2A

class Greeting {
  String say() => 'Hello from class';
}

extension GreetingExt on Greeting {
  String say() => 'Hello from extension';
}

// Extension nullable agar method bisa dipanggil ketika objek bernilai null
extension GreetingNullableExt on Greeting? {
  String sayNullable() {
    if (this == null) return 'Hello from extension (Null Object)';
    return 'Hello from extension (Not Null)';
  }
}

void main() {
  Greeting? obj = Greeting();
  
  print('--- Tes Pemanggilan Implisit ---');
  // Memanggil method asli class karena class method memiliki prioritas lebih tinggi
  print('obj?.say(): ${obj?.say()}'); // Output: Hello from class
  
  print('\n--- Memastikan Extension Method yang Dipanggil (Explicit Call) ---');
  // Cara memaksa memanggil extension method:
  print('GreetingExt(obj!).say(): ${GreetingExt(obj!).say()}'); // Output: Hello from extension
  
  print('\n--- Tes Objek Bernilai Null ---');
  obj = null;
  print('obj?.say() saat null: ${obj?.say()}'); // Output: null
  
  // Menggunakan extension nullable
  print('GreetingNullableExt(obj).sayNullable(): ${GreetingNullableExt(obj).sayNullable()}');
}
ANALISIS JAWABAN SOAL 3:

1. Apa output dari program asli?
Outputnya adalah: Hello from class.

2. Mengapa demikian?
Dart menetapkan aturan bahwa instance method (method asli kelas) selalu memiliki prioritas lebih tinggi daripada extension method jika keduanya memiliki nama dan signature parameter yang persis sama. Compiler akan selalu mendahulukan method yang didefinisikan langsung di dalam kelas.

3. Jika obj bernilai null, apa yang terjadi?
Pemanggilan obj?.say() akan menghasilkan nilai null dan program tidak akan mengalami crash karena operator null-aware (?.) menghentikan evaluasi method bila receiver bernilai null.

4. Bagaimana cara memastikan extension method yang dipanggil?
Bungkus objek target secara eksplisit menggunakan nama extension-nya: GreetingExt(obj!).say(). Kita harus melakukan null assertion (!) terlebih dahulu jika objek bertipe nullable, atau buat extension yang menargetkan tipe nullable (Greeting?) lalu panggil secara eksplisit.
